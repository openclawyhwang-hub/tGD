# Environment Variables

Complete list of environment variables used by Agentic-PDLC-Workflow.

## Jira Integration

Required for task management, ticket creation, and workflow automation.

| Variable | Description | Example |
|----------|-------------|---------|
| `JIRA_URL` | Jira instance URL | `https://your-domain.atlassian.net` |
| `JIRA_USER` | Jira account email | `you@company.com` |
| `JIRA_API_TOKEN` | Jira API token | Generated from Atlassian account settings |
| `JIRA_PROJECT_KEY` | Project key for ticket creation | `PROJ` |
| `JIRA_AI_ACCOUNT_ID` | Jira account ID for AI agent assignment | Found in Jira user profile URL |

**Setup:**
```bash
export JIRA_URL="https://your-domain.atlassian.net"
export JIRA_USER="you@company.com"
export JIRA_API_TOKEN="your-api-token"
export JIRA_PROJECT_KEY="PROJ"
export JIRA_AI_ACCOUNT_ID="jira-ai-user-id"
```

## Git Provider

Required for PR/MR creation and code review automation.

### GitHub (via `gh` CLI)

| Variable | Description | Example |
|----------|-------------|---------|
| `GITHUB_TOKEN` | Personal access token | `ghp_xxxxxxxxxxxx` |

**Setup:**
```bash
export GITHUB_TOKEN="ghp_xxxxxxxxxxxx"
gh auth login  # Or use `gh auth login` interactively
```

### GitLab (via `.git-provider.json`)

Create `.git-provider.json` in your repo root:

```json
{
  "provider": "gitlab",
  "api_url": "https://gitlab.your-company.com/api/v4",
  "project_id": "12345",
  "token_env": "GITLAB_TOKEN"
}
```

| Variable | Description | Example |
|----------|-------------|---------|
| `GITLAB_TOKEN` | GitLab personal access token | `glpat-xxxxxxxxxxxxxxxxxxxx` |

**Setup:**
```bash
export GITLAB_TOKEN="glpat-xxxxxxxxxxxxxxxxxxxx"
```

### Bitbucket (via `.git-provider.json`)

```json
{
  "provider": "bitbucket",
  "api_url": "https://api.bitbucket.org/2.0",
  "workspace": "your-workspace",
  "repo_slug": "your-repo",
  "token_env": "BITBUCKET_TOKEN"
}
```

| Variable | Description | Example |
|----------|-------------|---------|
| `BITBUCKET_TOKEN` | Bitbucket app password | Generated from Bitbucket settings |

**Setup:**
```bash
export BITBUCKET_TOKEN="your-app-password"
```

## CI/CD

Optional — for automated testing and deployment pipelines.

| Variable | Description | Example |
|----------|-------------|---------|
| `CI` | CI environment flag | `true` (set automatically by CI systems) |
| `CI_DB_PASSWORD` | Database password for integration tests | Generated by CI secrets |
| `DEPLOY_TOKEN` | Deployment authentication token | Provider-specific |
| `DOCKER_REGISTRY_TOKEN` | Container registry authentication | Provider-specific |

**GitHub Actions:**
```yaml
env:
  CI_DB_PASSWORD: ${{ secrets.CI_DB_PASSWORD }}
```

**GitLab CI:**
```yaml
variables:
  CI_DB_PASSWORD: $CI_DB_PASSWORD  # Set in CI/CD Settings → Variables
```

## Security Best Practices

1. **Never commit `.env` files** — Add `.env` to `.gitignore`
2. **Use CI secrets** — Store sensitive values in GitHub Secrets / GitLab CI Variables
3. **Rotate tokens regularly** — Update API tokens every 90 days
4. **Least privilege** — Give tokens only the permissions they need
5. **Audit access** — Review who has access to tokens quarterly

```bash
# Test Bitbucket connection
curl -u "your-username:$BITBUCKET_TOKEN" https://api.bitbucket.org/2.0/user
```
