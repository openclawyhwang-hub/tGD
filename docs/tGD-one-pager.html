A 7-stage pipeline that forces AI agents to read before writing, spec before coding, test before claiming done.
Every stage has explicit inputs, outputs, and a human sign-off gate. No shortcuts. No "should work". Only evidence.
1
🗺️ Map
/tgd-map
Input
Project source code (any repo)
Output
CONTEXT.md — tech stack, architecture, repo inventory, dependencies
🤖 Agent Scans codebase, initializes CodeGraph knowledge graph, maps patterns
2
📋 Define
/tgd-define
Input
CONTEXT.md + user intent (natural language)
Output
PRD.md SPEC.md DESIGN.md (optional) + feature branch
👤 PM Interviewed by agent → picks feature name → reviews & signs off PRD + SPEC
3
📐 Plan
/tgd-plan
Input
CONTEXT.md PRD.md SPEC.md
Output
TASKS.md — atomic tasks with acceptance criteria + regression markers
🤖 Agent Triple-source decomposition (CONTEXT + PRD + SPEC) 👤 DEV Reviews & signs off task breakdown
4
⚡ Develop
/tgd-develop
Input
TASKS.md SPEC.md + existing codebase
Output
src/ implementation + tests/ TDD test files (worktree-isolated)
🤖 Agent Implements in Git Worktree sandbox. <3 tasks → fast mode, ≥3 → parallel subagents
5
🧪 Verify
/tgd-verify
Input
src/ tests/ + REGRESSION-CATALOG.md
Output
TEST-REPORT.md — all tests + browser E2E + cross-feature regression
🤖 Agent Runs ALL tests + browser E2E + cross-feature regression. ANY failure → pipeline blocked.
6
🔎 Review
/tgd-review
Input
src/ tests/ TEST-REPORT.md
Output
REVIEW.md — 5-axis quality report (correctness · maintainability · security · perf · coverage)
🤖 Agent 5-axis code review (Correctness · Maintainability · Security · Performance · Coverage)
7
🚀 Ship
/tgd-ship
Input
Signed-off artifacts: PRD TASKS TEST-REPORT REVIEW
Output
CHANGELOG.md REGRESSION-CATALOG.md ADR.md + CI/CD deploy
👤 PM Final sign-off gate (all roles must approve) 🤖 Agent Atomic commits → CI → staged rollout → monitoring

🔄 tGD vs Loop Engineering

  • Same foundation — worktrees, skills, sub-agents, persistent memory
  • tGD adds Hard Gates — Verify blocks if ANY test fails, no exceptions
  • tGD enforces Sign-off — PM · DEV · QA must approve before Ship
  • tGD is auditable — full artifact trail: CONTEXT → PRD → SPEC → TASKS → TEST-REPORT → REVIEW → CHANGELOG

🧪 Three-Stage Testing

  • Builder (Develop) — Write tests alongside code, Red → Green → Refactor
  • Inspector (Verify) — Run ALL tests + browser E2E, block on failure
  • Auditor (Review) — Audit quality: 80% unit / 15% integration / 5% E2E
