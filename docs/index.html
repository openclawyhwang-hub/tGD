The Agentic PDLC Harness

tGD

Your AI agent wrote 500 lines of code. Did it run the tests? Read your codebase? Write a spec? Probably not. tGD is an 8-stage pipeline that forces agents to follow the same workflow you would — Map → Define → Plan → Build → Verify → Review → Ship. No shortcuts. No "should work". Just evidence. 你的 AI agent 寫了 500 行 code。它有跑測試嗎？讀過你的 codebase？寫過 spec 嗎？ 八成沒有。tGD 是一個 8 階段的 pipeline，強制 agent 跟你一樣走完整流程 — Map → Define → Plan → Build → Verify → Review → Ship。沒有捷徑。不准「應該能用」。只看證據。 AI エージェントが500行のコードを書いた。テストは走ったか？コードベースは読んだか？スペックは書いたか？ たぶん書いてない。tGD は8段階のパイプライン。エージェントにあなたと同じワークフォローを強制する — Map → Define → Plan → Build → Verify → Review → Ship。 shortcuts なし。「動くはず」なし。証拠だけ。 Dein AI-Agent hat 500 Zeilen Code geschrieben. Hat er die Tests laufen lassen? Deine Codebase gelesen? Ein Spec geschrieben? Wahrscheinlich nicht. tGD ist eine 8-Stufen-Pipeline, die Agenten zwingt, denselben Workflow zu befolgen wie du — Map → Define → Plan → Build → Verify → Review → Ship. Keine Abkürzungen. Kein "sollte funktionieren". Nur Beweise.

Why tGD?為什麼需要 tGD？ なぜ tGD なのか？ Warum tGD?

How Skills WorkSkills 如何運作 スキルの仕組み So funktionieren Skills

Each skill is a 4-part contract. Progressive disclosure — agents load details only when needed.每個 skill 都是四部分合約。漸進式揭露——agent 只在需要時載入細節。 各スキルは4部構成の契約。プログレッシブディスクロージャ——エージェントは必要な時だけ詳細をロード。 Jeder Skill ist ein Vertrag aus 4 Teilen. Progressive Disclosure – Agent lädt Details nur bei Bedarf.

01

Frontmatter

Name, description, triggers. Agent decides relevance in milliseconds.名稱、描述、觸發條件。Agent 毫秒內判斷相關性。名前・説明・トリガー。エージェントがミリ秒で関連性を判断。Name, Beschreibung, Trigger. Agent entscheidet Relevanz in Millisekunden.

02

Workflow

Numbered steps with exact commands. No ambiguity, no guessing.編號步驟加上精確指令。沒有模糊空間，不用猜。番号付きステップと正確なコマンド。曖昧さなし、推測なし。Nummerierte Schritte mit exakten Commands. Keine Mehrdeutigkeit, kein Raten.

03

Verification

Exit criteria. The agent must prove it worked before moving on.退出標準。Agent 必須證明有效才能繼續。終了条件。エージェントは動いたことを証明してから次に進む。Exit-Kriterien. Der Agent muss beweisen dass es funktioniert bevor es weitergeht.

04

Anti-rationalization

Blocks lazy shortcuts. "I'll just quickly implement this" → denied.擋住懶惰捷徑。「我快速實作一下就好」→ 不准。手抜きショートカットをブロック。「ちょっとサクッと実装するね」→ 却下。Blockt faule Abkürzungen. "Ich implementier das schnell" → verweigert.
