Why tGD?為什麼需要 tGD？ なぜ tGD なのか？ Warum tGD?

The problem isn't that agents can't code. It's that nobody holds them accountable.問題不是 agent 不會寫 code，而是沒有人管得住它。 問題はエージェントがコードを書けないことではない。誰も責任を持たないことだ。 Das Problem ist nicht, dass Agenten nicht coden können. Es ist, dass niemand sie zur Rechenschaft zieht.

FAQ常見問題 よくある質問 FAQ

Common questions about tGD.關於 tGD 的常見問題。 tGDに関するよくある質問。 Häufig gestellte Fragen zu tGD.

Do I need to install anything besides the agent?
Just bash setup.sh. It auto-detects your CLI and configures everything. Webwright dependencies installed automatically.
What if my agent doesn't support slash commands?
Say "Plan this feature" in natural language — tGD maps intent to skills automatically.
Can I skip stages?
Each stage has pre-flight checks. If you skip, the next stage will block you.
Does this work with existing projects?
Yes! /tgd-map scans your existing codebase first.
How is this different from just using Cursor/Copilot?
Those tools write code. tGD enforces a workflow — specs, plans, tests, reviews — before code ships.
Can I customize the pipeline?
Yes! Edit the skill files in skills/ to match your team's workflow.
Will this slow down my agent?
No. Skills load on-demand — only the relevant skill enters context. ~5% overhead per skill. Most of the time, your agent works at full speed.
Does it work offline?
Core skills work without internet. E2E testing with Webwright needs a browser, but spec writing, planning, code review, and TDD are all offline-capable.
Can I use multiple CLIs?
Yes. The same 26 skills work across Claude Code, Codex, Gemini CLI, OpenCode, and Pi. Commands auto-detect your platform.
